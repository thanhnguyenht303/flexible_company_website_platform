# i18n

The platform supports English and Vietnamese UI text and bilingual content fields for major content modules.

## UI Translations

UI copy is handled through the language provider and translation files under `lib/i18n`. Admin and public components should use translation keys instead of hard-coded labels when the text appears in the interface.

Language switching is available in public/admin UI areas, and stored preferences are reused on later visits.

## Content Field Convention

Localized content follows paired English/Vietnamese fields:

```text
name / nameVi
summary / summaryVi
description / descriptionVi
title / titleVi
excerpt / excerptVi
content / contentVi
position / positionVi
bio / bioVi
```

When Vietnamese content is missing, public rendering should fall back to the English field instead of showing blank content.

## Modules With Localized Content

Current bilingual content appears in:

- Site and theme-facing labels where translated through UI dictionaries.
- Products and services.
- Posts and article content.
- Team member position/bio fields.
- Page-builder text content where admins provide translated copy through content fields or duplicated page content strategy.
- Email templates through per-language template records.

## Vietnamese Terminology

Keep Vietnamese admin labels consistent across the product. Recommended terms:

| English | Vietnamese |
| --- | --- |
| Forms | Biểu mẫu |
| Leads | Khách hàng tiềm năng |
| Q&A | Hỏi đáp |
| Email Center | Trung tâm email |
| Roles | Vai trò |
| Authorities | Quyền |
| Site Settings | Cài đặt website |
| Theme | Giao diện |
| Page Builder | Trình dựng trang |

Use the existing translation file wording if it differs; update the glossary and UI together when changing terms.

## Adding New Translated UI

1. Add keys to the English and Vietnamese translation dictionaries.
2. Use the existing translation hook/helper in React components.
3. Avoid hard-coded English strings in admin buttons, empty states, validation copy, and table headers.
4. For API validation messages, prefer stable error codes plus UI-side translation when the message is shown to users.
5. Verify text wrapping in narrow admin and mobile public layouts.

## Email Templates

Email templates are language-specific through `EmailTemplate.language`. Variable names stay the same across languages:

```text
{{applicantName}}
{{leadEmail}}
{{questionTitle}}
```

Translate the visible subject/body text, not the variable keys.
